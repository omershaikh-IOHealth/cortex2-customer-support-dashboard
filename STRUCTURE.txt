API ENDPOINTS (Backend)
=======================

/api/health                           # Health check
/api/metrics/overview                 # Dashboard metrics
/api/tickets                          # All tickets (with filters)
/api/tickets/:id                      # Single ticket + threads
/api/sla/critical                     # Critical SLA tickets
/api/escalations                      # All escalations
/api/logs                             # System logs
/api/analytics/trends                 # 30-day trends
/api/analytics/priority-distribution  # Priority stats
/api/config/sla                       # SLA configuration
/api/config/escalation                # Escalation rules


PAGES & ROUTES (Frontend)
==========================

/                      → Redirects to /dashboard
/dashboard            → Main overview with metrics
/tickets              → Ticket list with filters
/tickets/:id          → Full ticket details
/sla                  → SLA monitoring dashboard
/escalations          → Escalation alerts
/analytics            → Charts and trends
/logs                 → System execution logs
/config               → System configuration


KEY TECHNOLOGIES
================

Backend:
- Express.js          → API server
- PostgreSQL (pg)     → Database client
- WebSockets (ws)     → Real-time updates
- CORS                → Cross-origin requests

Frontend:
- Next.js 14          → React framework
- React Query         → Data fetching/caching
- Tailwind CSS        → Styling
- Recharts            → Data visualization
- Lucide Icons        → Icon library
- date-fns            → Date formatting


DEVELOPMENT WORKFLOW
====================

1. Start Backend:     cd backend && npm run dev
2. Start Frontend:    cd frontend && npm run dev
3. Access:            http://localhost:3000
4. API Docs:          http://localhost:5000/api/health
